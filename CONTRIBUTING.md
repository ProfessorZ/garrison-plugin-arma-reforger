# Contributing

Contributions are welcome! Follow this workflow to submit changes.

## Fork and PR Workflow

1. **Fork** this repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/<your-username>/garrison-plugin-arma-reforger.git
   cd garrison-plugin-arma-reforger
   ```
3. **Create a branch** for your change:
   ```bash
   git checkout -b my-feature
   ```
4. **Make your changes** and commit with a clear message:
   ```bash
   git add .
   git commit -m "Add my feature"
   ```
5. **Push** your branch to your fork:
   ```bash
   git push origin my-feature
   ```
6. **Open a Pull Request** against the `main` branch of this repository.

## Guidelines

- Keep PRs focused on a single change.
- Follow existing code style and conventions.
- Test your changes against a running Arma Reforger server if possible.
- Update documentation if your change affects usage or configuration.
